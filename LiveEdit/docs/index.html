Live Edit Documentation

Overview

Live Edit is a tool that allows editing of your site content directly on your website. It brings some of the backend content management functions to the frontend. Depending on the Live Edit functionality enabled on the site and your permissions, you can inspect meta data, edit, rearrange, remove, and add new content on your site.

Browser Support

Here are the current support browsers:

  • Google Chrome Desktop
  • Safari Desktop
  • Safari Mobile (limited functionality)
  • Firefox Desktop
  • Internet Explorer 11+ Desktop
  • Internet Explorer 10 Desktop (limited functionality, unsupported)

Support for other browsers and devices are planned in the future.

How To Activate Live Edit

There are two ways you can activate Live Edit. You can be editing a page in VipaSuite and click the Live Edit button in the upper right-hand corner to open the page with Live Edit activated. You can also click the Live Edit button in header of the navigation side-bar, which would open your site's home page and activate Live Edit.

Login Required

If you happen to refresh your page after inactivity (about one hour), while Live Edit is activated, you will get the message in the screenshot below to login into VipaSuite.

The User Interface

The user interface has 3 main interface elements: the primary toolbar, the component toolbar, and the component selector. The primary toolbar is the main interface for the application. It is used to select CMS components in order to inspect, edit, and rearrange content. Since the primary toolbar sits on top of the website, it can be moved and minimized so you can access the content underneath it. Once a component is selected, a component toolbar is shown next to where the component is on the web page.

Component Selector

The component selector is used to search for components to add to the web page.

The Primary Toolbar

The toolbar provides the following functions:
  • Adding content to the page via drag and drop.

  • Navigating through the components on the page: out (to the parent level), previous, and next. Navigation is how you select components in order to make changes to your website. You can also use the mouse in combination with a modifier key (CTRL, ALT, CMD, or SHIFT) to directly navigate to a component.

  • View of basic component information: icon, name, and type. If the component is on the page template or shared, you'll see the template icon, T, or shared icon, r, as well. This information is useful when making a decision to move, edit, or remove a component.

  • Actions for the selected component.

    • m - move the selected component to another place on the page.
    • e - edit the selected component.
    • x - remove the selected component from either the page or page template. You can tell which it will be removed from by the T icon.
    • i - view advanced information about the selected component.

  • Controls for the toolbar.

    • ? - help page.
    • t / b - location toggle: top or bottom.
    • ⌃ / ⌄ - visibility toggle: shown or hidden.
    • q - quit the Live Edit application.

The Component Toolbar

The component toolbar provides basic information about the selected component as well as buttons for the available actions: move, edit, add, and remove. Which actions are available depends on the configuration of the site as well as your permissions.

The toolbar also shows you where the component is within the page. You can use the navigation buttons in the primary toolbar to navigate to a specific component or you can directly navigate with your mouse. Mouse navigation requires that you hold down a modifier key (CTRL, ALT, CMD, or SHIFT) when clicking on an area of the page.

If the component is on the page template or shared, you'll see the template icon, T, or shared icon, r. This information is useful when making a decision to move, edit, or remove a component, since these icons indicate that other pages would be affected in addition to the one you are viewing.

Finally, you can navigate to components that are not currently visible on the page. This commonly occurs with dynamic content that automatically displays at some interval or due to user interaction like a drop down menu. When you select a component like this, you'll see some text to indicate that the component was not visible when selected.

Keyboard Shortcuts

There are several keyboard shortcuts available for the Live Edit application. Keyboard shortcuts must be combined with a modifier key (CTRL, ALT, CMD, or SHIFT). Any combination of one or more modifier keys can be used. Sometimes you will need to use more than one modifier if a shortcut conflicts with a browser or desktop shortcut.

Commonly used modifiers by operation system and browser:

Windows

  • Chrome - ALT.
  • Firefox - ALT+SHIFT.

Mac

  • Chrome - CMD.
  • Safari - CMD.
  • Firefox - CMD+ALT. ALT is the same as OPTION.
Sometimes you will need to add other modifier keys. For example, CMD+V is a shortcut for paste on the Mac. To toggle visibility you could use CMD+ALT+V instead of CMD+V to avoid the conflict

Linux

  • Chrome - CTRL.
  • Firefox - CTRL+ALT.
Shortcuts

Must be used with one or more of the modifiers mentioned above. Uppercase or lowercase letters are equivalent (e.g. "x" and "X" work the same).

  • Navigation - Arrow Keys.
  • Open Add Component Interface - "X".
  • Toggle Visibility Of Primary Toolbar - "V".
  • Toggle Location Of Primary Toolbar - "L".
  • Toggle Information Popup In Primary Toolbar - "I".
  • Open Help Documentation (This Page) - "H".
